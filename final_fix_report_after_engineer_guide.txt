3. 未修复漏洞清单（需要进一步方案）
--------------------------------------------------------------------------------

  【Critical - 未修复 0 个】
  当前已无遗留 Critical 漏洞。

  【High - 未修复 6 个】

  H-1   ollama 非可选依赖
        原因: src/router.py 顶部仍 import ollama；改为方法内导入会影响类型检查、
              测试 patch 路径及大量现有代码，需要系统性调整导入与错误处理。
        所需方案: 确认是否接受 ollama 作为可选依赖，以及缺失时的降级行为。

  H-4   长时间推理无进度反馈（完全版）
        原因: 完整方案需要接入 ollama.chat(..., stream=True) 或定期进度打印，
              会改变 route() 返回值/交互方式。
        所需方案: 确认是否增加流式输出 API 或保持当前阻塞式接口。

  H-11  reset_stats 无防篡改
        原因: 需要引入调用者权限校验或审计日志，当前项目尚无权限抽象。
        所需方案: 给出防篡改策略（如密码保护、审计日志、仅内部调用）。

  H-23  架构劣化（God Class）
        原因: 拆分 SmartRouter 为 InferenceBackend / RoutingStrategy / StatsCollector /
              TimeoutService / ModelCache 属于大型重构，远超当前修复范围。
        所需方案: 提供目标包结构与迁移计划。

  H-24  策略模式名存实亡
        原因: 同 H-23，需要抽象策略接口并迁移硬编码 if/elif 路由逻辑。
        所需方案: 与 H-23 一并规划。

  H-25  无配额管理 / H-27 无速率限制
        原因: 需要按用户/会话/token 设计配额与限流，当前项目无用户概念；
              实现会改变并发模型并需要持久化或共享状态。
        所需方案: 给出配额/限流维度（全局/IP/用户）与算法选择。
